The Server Saga

Some people just want a simple server on the cheap, and are not willing to pay for premium support for it. This article will show you how to run a server with only paying for Internet, Electricity, and the Hardware. In this article, you will be installing Ubuntu GNU/Linux, running a website, cloud storage, and remote access.

Part 1: Picking the right hardware

Before you start, you must consider the question: What are you going to use the server for? In this article , we are going to be installing a Web Server (website), WebDAV (cloud storage), and a Database Server (back-end). These three combined have very little load on the actual hardware, unless you are planning to have over 50 concurrent users. With about 10 concurrent users at the same time, you can get away with a 2014 mid-range laptop. I do not recommend using a Raspberry Pi for this project (although it can be done) because we are going to be using Docker containers. After everything is installed and running, the server is going to be using at least 1.5 GB of RAM at idle, which a Raspberry Pi cannot give.

Minimum specs:

  • 4GB RAM

  • Dual Core Processor (Turbo up to 2.1 gigahertz)

  • 20GB Internal Drive

  • External HDD/SSD (for cloud storage)

Recommended specs:

  • 8GB RAM

  • Quad Core Processor (Turbo up to 3.4 gigahertz)

  • 60GB Internal SSD

  • 2+ TB External HDD

Part 2: Installing Ubuntu Server

Every server must have a robust, reliable operating system to run off of. In this case, it is Ubuntu Server 18.04. I chose 18.04 over everything else because 18.04, at the time of writing, is the latest LTS (long term support) release. This means that it is much more stable than 18.10 and better than 16.04 (the previous LTS release). First, get an installation ISO (here) and flash it to a USB drive of at least 1GB. For instructions on how to do that, see this article. The process of installing Ubuntu Desktop to a USB drive is very similar to that of Ubuntu Server. Once that is done, boot from the USB. If you are having difficulty with that, see this article. Follow the prompts on screen to install Ubuntu Server.

Part 3: Installing SSH

In this part, we will install the SSH. SSH (Secure SHell) is the de facto standard for remote connections to *nix operating systems. This is usually served over port 22 and will need to be port forwarded (if you want to remotely connect outside of your LAN)

Installing SSH is a very basic command. Run

$ sudo bash -c "apt update && apt install openssh-client openssh-server"

This will install the libraries needed for running an SSH server.

To connect to your server from the LAN, find out the local IPv4 address of the machine. Run this command on the server:

$ ip a

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1

    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00

    inet 127.0.0.1/8 scope host lo

       valid_lft forever preferred_lft forever

    inet6 ::1/128 scope host

       valid_lft forever preferred_lft forever

6: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP group default qlen 1000

    link/ether ff:ff:ff:ff:ff:ff brd ff:ff:ff:ff:ff:ff link-netnsid 0

    inet 192.168.1.1/30 brd IPADDRESS scope global arc0

       valid_lft forever preferred_lft forever

    inet6 fe80::2ff:aaff:fe00:55/64 scope link

       valid_lft forever preferred_lft forever

 

To connect from a Linux or Mac client, run this command in the terminal. Linux users may have to install openssh-client, but the instructions are different for each distribution. On a Mac, it may prompt you to install developer tools. Do that. Run:

$ ssh username@IPADDRESS

Enter password for username:

username@hostname$

 

You now have the same prompt on this computer, and do not have to be physically at the server to run commands.
